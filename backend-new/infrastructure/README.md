# PashuVaani Infrastructure & Deployment Guide

This folder contains the Terraform configuration to provision the AWS cloud infrastructure for the PashuVaani Backend.

## 🚀 Quick Start

### 1. Prerequisites
- **AWS CLI** configured with credentials.
- **Terraform** installed.
- Your public IP added to `security_groups.tf` (if connecting from home).

### 2. Infrastructure Setup
```bash
# Navigate to infrastructure folder
cd infrastructure

# Initialize backend (S3 & DynamoDB)
terraform init -reconfigure `
  -backend-config="access_key=YOUR_KEY" `
  -backend-config="secret_key=YOUR_SECRET"

# Apply changes
terraform apply -auto-approve
```

### 3. Deploy Application (Docker)
1. Ensure `.env` is updated with the `rds_endpoint` from Terraform output.
2. Run the deployment sequence:
```bash
# SSH into EC2
ssh -i infrastructure/deploy_key.pem ubuntu@<EC2_IP>

# Inside VM
cd ~/backend-docker
sudo docker-compose up -d --build
```

## 🛠 Management
- **SSH Access**: `ssh -i infrastructure/deploy_key.pem ubuntu@<EC2_IP>`
- **API Docs**: `http://<EC2_IP>/docs`
- **RDS Connectivity**: Endpoint: `...rds.amazonaws.com`, User: `dbadmin`, Port: `5432`

## 🔒 Security Note
The `deploy_key.pem` is generated by Terraform and stored locally for SSH access. Keep this file secure and do not commit it to version control.
